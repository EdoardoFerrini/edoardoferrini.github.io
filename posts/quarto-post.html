Personalizzazioni Blog — Quartz 5.0

Questo documento descrive tutte le modifiche e i plugin custom realizzati per trasformare Quartz 5.0 in un blog funzionante con elenco post, tag e integrazioni social.

1. Architettura

Quartz 5.0 usa un sistema a plugin categorizzati in:

CategoriaRuolo
transformerElabora il Markdown → AST
filterFiltra i file (es. bozze)
emitterGenera file HTML/output
pageTypeDecide COME viene renderizzata ogni pagina
componentFrammenti UI (senza logica di processing)

I 5 plugin custom sostituiscono/integrano i plugin built-in per ottenere un comportamento da blog.

2. Plugin Custom

Tutti i plugin live in plugins/ e sono collegati a .quartz/plugins/ tramite junction (equivalenti a symlink su Windows, creati automaticamente da npx quartz plugin install).

2.1 blog-link

Posizione: Header — icona GitHub + RSS

plugins/blog-link/
├── index.js          # Entry point
├── components.js     # Componente Preact con SVG inline
└── package.json
  • Mostra le icone GitHub (link a https://github.com/edoardoferrini) e RSS (link a /atom.xml)
  • Usa SVG inline, nessuna dependency esterna
  • Nessun build step necessario

2.2 posts-list

Posizione: afterBody — Elenco cronologico dei post in stile blog

plugins/posts-list/
├── package.json
└── dist/
    ├── index.js        # Entry point
    └── components.js   # Componente compilato

Comportamento per pagina:

PaginaSlugMostra
Homepage"" o "index"Ultimi 5 post + link “tutti i post →“
Posts"posts/index"Tutti i post (nessun link “vedi tutti”)
Tag specifico"tags/..."Solo post con quel tag, filtrati
Altre pagine—Non viene renderizzato (return null)

Ordinamento: Data discendente (modified → created → frontmatter.date) Locale: Usa cfg.locale dal config (it-IT)

Per modificarlo bisogna ricompilare il source. Vedi Manutenzione.

2.3 tags-list

Posizione: afterBody — Tag cloud con conteggio post

plugins/tags-list/
├── package.json
└── dist/
    ├── index.js        # Entry point
    └── components.js   # Componente compilato
  • Mostra tutti i tag unici con conteggio post (es. test (3))
  • Visibile solo in homepage — nascosto su /posts/ e /tags/
  • Tag ordinati alfabeticamente
  • Link “tutti i tags →” a /tags/
  • Per modificarlo bisogna ricompilare il source.

2.4 tag-page-custom

Tipo: pageType + component — Pagine tag (sostituisce github:quartz-community/tag-page)

plugins/tag-page-custom/
└── index.js

Cosa fa:

  • Matcha slug che iniziano con tags/
  • Genera automaticamente pagine virtuali per ogni tag presente nei post (incluse gerarchie: a/b → genera sia a che a/b)
  • Pagina /tags/: elenca tutti i tag con link
  • Pagina /tags/<nome>: mostra intro “#tag” + usa posts-list in afterBody per i post filtrati
  • Usa layout "content", stesso layout delle pagine normali

2.5 folder-content-page

Tipo: pageType — Pagine di cartella senza lista file (sostituisce github:quartz-community/folder-page)

plugins/folder-content-page/
└── index.js

Perché esiste: Il folder-page built-in aggiunge automaticamente una lista di file (PageList) nella parte superiore della pagina. Con posts-list già presente, si creava duplicazione.

Cosa fa:

  • Matcha slug che finiscono con /index (es. posts/index)
  • Esclude homepage (index) e pagine tags (gestite da tag-page-custom)
  • Renderizza solo il contenuto Markdown del file index.md
  • Nessuna lista file automatica

5. Flusso di Navigazione

Richiesta browser          Page Type matchato          Cosa viene renderizzato
─────────────────          ──────────────────          ───────────────────────
/                           ContentPage                  Homepage (index.md)
                                                         + posts-list (5 post)
                                                         + tags-list

/posts/                     FolderContentPage            posts/index.md (contenuto)
                                                         + posts-list (tutti)

/posts/primo-post           ContentPage                  primo-post.md

/tags/                      TagPageCustom                Elenco tag (tutti i tag)
                                                         + posts-list (nessun post)

/tags/test                  TagPageCustom                Intro #test
                                                         + posts-list (filtrati)

Come il server dev gestisce gli URL

Quando richiedi /posts/ (con slash finale):

1. Il server controlla se esiste public/posts/index.html → SÌ
2. Lo serve come pagina HTML
3. body[data-slug] = "posts/index" (impostato al build time)

Modificare i componenti compilati (posts-list, tags-list)

I componenti in plugins/*/dist/ sono precompilati. Per modificarli:

  1. Trova il source originale (potrebbe essere in un repo npm o in plugins/*/src/)
  2. Ricompila con: cd plugins/<nome> && npm run build
  3. Poi: npx quartz build

In alternativa, riscrivi il componente in JS puro (come blog-link) e rimuovi il dist/.
