Hostare su GitHub Pages gratuite da un repository privato

10 maggio 2026
tutorialgithubgithub-actions

Una guida passo passo per usare GitHub Actions e pubblicare un sito statico su GitHub Pages partendo da un repository sorgente privato.

Indice
  • 1. Prepara i due repository
  • 2. Genera il “Passepartout” (Personal Access Token)
  • 3. Salva il Token nel Repo Privato
  • 4. Il tuo primo Deploy
  • 5. Ultimo step: Attiva Pages

Oggi vediamo come configurare un workflow per fare il deploy su GitHub Pages quando il nostro codice sorgente si trova in un repository privato, aggirando il limite dei repository privati sulle versioni gratuite di GitHub.

Invece di usare l’azione standard (che blocca il deploy se il repo è privato), useremo peaceiris/actions-gh-pages. Questa action esegue la build nel repo privato e trasferisce solo i file HTML/CSS/JS finali verso un repository pubblico a tua scelta (ad esempio Username/my-project).

Ecco i passaggi esatti che devi seguire su GitHub per far funzionare il tutto:

1. Prepara i due repository

  1. Vai nel tuo attuale repository su GitHub (es. Username/my-project) dove tieni il codice sorgente o i file del tuo sito.
  2. Vai in Settings, rinomina il repo in my-project-source (o my-project-private) e poi, in fondo nella sezione Danger Zone, rendilo Private.
  3. Torna alla home di GitHub e crea un Nuovo Repository vuoto.
  4. Chiamalo my-project, assicurati che sia Public e NON inizializzarlo con README o .gitignore. Lascialo vuoto.

2. Genera il “Passepartout” (Personal Access Token)

Dato che i due repository sono separati, le GitHub Actions hanno bisogno di un permesso speciale per poter scrivere dal repo privato a quello pubblico.

  1. Clicca sulla tua foto profilo in alto a destra su GitHub e vai in Settings.
  2. Scorri in fondo a sinistra e clicca su Developer settings.
  3. Vai su Personal access tokens → Tokens (classic).
  4. Clicca Generate new token (classic).
    • Note: Deploy my-project (o un nome a tua scelta)
    • Expiration: No expiration
    • Scopes: Spunta solamente la casella principale repo (Full control of private repositories).
  5. Genera il token e copialo subito (non potrai più vederlo in seguito!).

3. Salva il Token nel Repo Privato

  1. Apri il tuo repository privato (my-project-source).
  2. Vai in Settings → Secrets and variables → Actions.
  3. Clicca su New repository secret.
    • Name: DEPLOY_TOKEN (Deve chiamarsi esattamente così, il file di workflow .yml lo cercherà con questo nome).
    • Secret: [Incolla il token copiato prima]
  4. Clicca Add secret.

4. Il tuo primo Deploy

A questo punto è tutto pronto! Non ti resta che fare il commit delle modifiche e spingerle online (assicurati di aver aggiornato il file .github/workflows/deploy.yml nel tuo progetto locale per usare il nuovo workflow):

git add .
git commit -m "Update deploy workflow per workaround repo privato"
git push

Questo avvierà la GitHub Action su my-project-source, la quale spingerà i file finali del sito nel nuovo repository pubblico my-project sul ramo main (o gh-pages).

5. Ultimo step: Attiva Pages

Dopo aver visto che la Action ha finito con successo:

  1. Vai nel repository pubblico (my-project). Vedrai che ora ci sono dei file!
  2. Vai in Settings → Pages.
  3. Sotto “Build and deployment”, come “Source” seleziona Deploy from a branch.
  4. Come ramo seleziona main (o gh-pages, se vedi quello) e clicca Save.

D’ora in poi potrai lavorare sul tuo progetto nel completo rispetto della tua privacy: il mondo vedrà solo il risultato finale all’indirizzo username.github.io/my-project!

(Nota: puoi ripetere la stessa identica procedura per qualsiasi framework statico o generatore di siti come Astro, Next.js, Hugo, ecc.).
