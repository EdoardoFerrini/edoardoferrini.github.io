Ho provato Jekyll, Hugo, Next.js e altri. Alla fine sono tornato ad **Astro**.

<Callout type="info" title="Perché Astro?">
  Astro è progettato per siti **content-first**: zero JavaScript nel browser
  per default, pagine veloci senza configurazione aggiuntiva.
</Callout>

## Cosa mi piace

**Islands architecture.** Puoi aggiungere componenti interattivi (React, Vue, Svelte) solo dove servono. Il resto è HTML statico.

**Content Collections.** Type-safe, con schema Zod, validazione automatica.

<Callout type="tip" title="Islands in pratica">
  Aggiungi `client:load` a un componente React solo se ha davvero bisogno
  di interattività. Il resto rimane HTML morto — e la pagina vola.
</Callout>

## Come iniziare

<Steps>
  <li>Crea il progetto con `npm create astro@latest`</li>
  <li>Aggiungi l'integrazione MDX con `npx astro add mdx`</li>
  <li>Crea i tuoi componenti in `src/components/`</li>
  <li>Scrivili nei post `.mdx` senza nessun import aggiuntivo</li>
</Steps>

<Callout type="warning" title="Attenzione ai .md esistenti">
  Se rinomini un post da `.md` a `.mdx`, elimina il file `.md` originale —
  avere entrambi causa un conflitto di ID nella content collection.
</Callout>

## Conclusione

Se scrivi un blog nel 2026, Astro è probabilmente la scelta più sensata. Veloce, flessibile, con un ottimo DX.
