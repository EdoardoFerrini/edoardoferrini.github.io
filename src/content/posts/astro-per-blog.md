Ho provato Jekyll, Hugo, Next.js e altri. Alla fine sono tornato ad **Astro**.

## Perché Astro?

Astro è progettato per siti content-first. Non ha JavaScript nel browser per default — il che significa pagine veloci senza configurazione aggiuntiva.

```bash
npm create astro@latest
```

## Cosa mi piace

**Islands architecture.** Puoi aggiungere componenti interattivi (React, Vue, Svelte) solo dove servono. Il resto è HTML statico.

**MDX.** Scrivi Markdown con componenti dentro. Perfetto per un blog tecnico.

**Content Collections.** Type-safe, con schema Zod, validazione automatica.

## Conclusione

Se scrivi un blog nel 2026, Astro è probabilmente la scelta più sensata. Veloce, flessibile, con un ottimo DX.
